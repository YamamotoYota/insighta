# INSIGHTA 配布版 README

このフォルダは、Windows 向け配布版 `INSIGHTA.exe` をそのまま実行するためのフォルダです。  
Python や仮想環境の準備は不要です。

## 1. このフォルダにあるもの

- `INSIGHTA.exe`
  - アプリ本体です。
- `README.md`
  - この説明書です。

## 2. 起動方法

1. `INSIGHTA.exe` をダブルクリックします。
2. 数秒待つと、既定ブラウザで既定の `http://127.0.0.1:8050` が開きます。
3. 画面が表示されたら利用開始できます。

補足:
- 初回起動時や PC 負荷が高い時は、起動まで 10〜30 秒程度かかることがあります。
- 自動でブラウザが開かない場合は、手動で `http://127.0.0.1:8050` を開いてください。
- 必要なら `INSIGHTA_HOST` / `INSIGHTA_PORT` 環境変数で接続先を変更できます。

## 3. 終了方法

- 画面上部の `アプリを終了` ボタンを押してください。
- 停止後はブラウザタブを閉じてください。

## 4. できること

- CSV / Excel 読み込み
- SQL データベース読み込み
- PI データ読み込み
  - PI DA タグ
  - PI AF 属性
  - PI AF イベントフレーム
- データテーブルとグラフの連動選択
- 散布図 / ヒストグラム / 箱ひげ図 / 散布図行列
- 単純移動平均 / 指数移動平均 / STL 分解
- 回帰 / 分類 / 時系列 / PCA / ICA / PCA 異常予兆検知（T2 / Q）
  - 時系列: ARIMA / SARIMA / EWMA / CUSUM
- モデル保存 / 再読込
- 加工後データの CSV / Excel 出力

## 5. データテーブルについて

データテーブルは、大きいデータでも扱いやすいように次の仕様です。

- フィルタ / ソート / ページ送りに対応
- 選択状態はページをまたいで維持
- `全選択` は現在のフィルタ結果全体に対して動作

補足:
- 以前のような全件一括スクロール表示ではなく、現在はページ単位で表示します。
- 大容量データでは、この方式の方が安定して動作します。

## 6. 動作環境

- Windows 64bit
- Edge / Chrome などのブラウザ

補足:
- SQL 接続では追加ドライバが必要な場合があります。
- PI データ取得では PI AF Client / AF SDK が必要です。
- PI 機能を使う配布物では `pythonnet` を含めてビルドしている必要があります。

## 7. よくあるトラブル

### 7.1 画面が開かない
- 数秒から 30 秒ほど待ってください。
- それでも開かない場合は、ブラウザで `http://127.0.0.1:8050` を直接開いてください。

### 7.2 `8050` ポートが既に使われている
- 既に別の INSIGHTA や他アプリが起動している可能性があります。
- 起動済みアプリを終了してから再実行してください。

### 7.3 Excel ファイルが読めない
- 配布版には Excel 読み込みライブラリを同梱しています。
- ファイル破損や拡張子不一致を確認してください。
- 50MB 級の Excel では読み込みに時間がかかる場合があります。

### 7.4 SQL 接続ができない
- DBMS ごとに OS 側ドライバや設定が必要な場合があります。
- SQL Server は ODBC Driver 17/18 for SQL Server を確認してください。

### 7.5 PI AF / PI DA が取得できない
- PI AF Client / AF SDK のインストールを確認してください。
- PI サーバー名、AF データベース名、エレメント名やパスの指定を確認してください。

## 8. セキュリティに関する補足

- 本アプリはローカル PC 上の `127.0.0.1` で動作します。
- 外部公開用サーバーではありません。
